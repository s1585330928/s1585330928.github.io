用户需求分析：先按搜索意图建站

玩家带着具体问题来搜索。下面按六类意图拆成「可承接的模块」：每块写明典型问法、本站如何落地、未来可拆成的详情页骨架，以及站内跳转，便于你按优先级补内容与做 SEO。

以下为栏目规划示意
1

新手入坑

意图是「别走弯路」：在信息爆炸里快速知道先做什么、别浪费什么，以及武器与玩法是否适合自己。

典型搜索意图

  • 刚下载 / 回流：先跟主线还是清支线？装备强化与精炼优先级？
  • 选武器：双持怎么配、近战远程手感、单人友好与组队缺口。
  • 资源：技能点、材料、合约与代币先投哪里；哪些坑会拖慢进度。

本站如何承接

  • 构筑库用「新手」筛选给出低操作成本的武器组合与技能表。
  • 每日待办把「上线先做三件事」写成可勾选清单，形成回访习惯。
  • 首页与 roadmap 标明「新手指南」扩展位，避免与构筑长文抢首屏。

详情页建议（SEO 骨架）

  • 总览页：版本入坑路线图（章节式：1～10 / 10～20 / 满级后）。
  • 子页：资源优先级表、常见误区 FAQ、单职业/武器生态简介。

数据与维护

  • 随版本更新「推荐构筑标签」与每日清单条目；外链官方补丁说明作佐证。
关键词池（可被首页搜索筛选）
新手指南开荒路线避坑资源规划
构筑库 · 新手筛选 每日待办 站点规划 · 内容扩展
2

构筑查询

意图是「抄作业可执行」：武器对、技能名、职责标签一眼对齐，减少论坛翻楼成本。

典型搜索意图

  • 按武器搜：弓匕、大剑匕首、法杖魔杖等具体搭配与变种。
  • 按场景搜：团本输出、野外单刷、战场、守点坦克、团血治疗。
  • 按游戏内用语搜：技能英文名 / 关键被动，对照本站表格。

本站如何承接

  • 构筑库：多标签筛选 + 双武器技能表（与游戏内命名对齐）。
  • URL 带 ?q= 便于分享「筛选后的列表」给队友。

详情页建议（SEO 骨架）

  • 单构筑长文：定位、优缺点、技能循环要点、装备与铭文方向（简表即可）。
  • 对比页：同职责两套构筑的差异（上手难度、资源依赖）。

数据与维护

  • 版本变动时批量更新 js/data.js 中构筑条目与标签；保留旧版备注或「已过期」提示。
关键词池
武器搭配PVEPVP技能表

3

副本机制

意图是「进本前心里有底」：每个 Boss 阶段要干什么、灭团点在哪、队伍最少配置是什么。

典型搜索意图

  • 按副本名 / Boss 名搜机制时间轴、地板技能、必打断技能。
  • 按职责搜：坦克拉怪位置、治疗抬血窗口、远程分散集合。
  • 开荒向：装备门槛、推荐构筑、常见灭团三次原因。

本站如何承接

  • 副本页先按类型聚合入口，再为热门本拆独立静态页（便于收录）。
  • 与构筑库交叉链接：该本推荐职责与武器标签。

详情页建议（SEO 骨架）

  • 单本攻略：总览 → 各 Boss 阶段表（技能名 / 应对 / 站位简图占位）。
  • 一页式速查：仅列灭团点与检查清单，供手机开打前扫一眼。

数据与维护

  • 机制以「可验证」为准：随补丁改时间轴时更新日期与版本脚注。
关键词池
机制速查站位队伍配置灭团点
副本入口 坦克构筑
4

Boss / 活动时间

意图是「对齐服务器时钟」：几点打什么、和谁抢、掉落与参与条件，最好一页能订阅或收藏。

典型搜索意图

  • 世界 Boss / Archboss：刷新段、参与人数、掉落与代币用途。
  • 限时活动与节日排期；跨时区玩家换算本地时间。
  • 与公会指挥表对齐：提前集合、分包、替补规则（外链 Discord 亦可）。

本站如何承接

  • Boss 与活动页用表格 + 说明文字承载时间表；后续可加「导出 .ics」或打印友好样式。
  • 与每日清单联动：活动日自动勾「提醒项」占位（静态站可用文案引导手动勾选）。

详情页建议（SEO 骨架）

  • 单 Boss 页：时间、地图、机制摘要、掉落表、推荐人数。
  • 「本周日程」聚合页：利于重复访问与收藏。

数据与维护

  • 时间类内容必须标注数据来源与最后校对日期；维护公告后第一时间改表。

时间表掉落提醒Archboss

5

公会 / PVP

意图是「组织战可执行」：谁站哪、什么构筑克制、资源点与指挥口令，比个人攻略更偏「流程与分工」。

典型搜索意图

  • 攻城 / 公会战：分队、破门顺序、投石与补给、替补轮换。
  • 野外据点：Boonstone、Riftstone 等目标争夺节奏与职业需求。
  • PVP 构筑与反制：针对主流组合的换武器与技能取舍。

本站如何承接

  • 静态站内以「索引 + 摘要」为主：链到构筑库 PVP 标签、Boss 时间表、每日清单中的公会项。
  • 长流程攻略可拆为多页，避免单页过长；指挥表可用外链表格补充。

详情页建议（SEO 骨架）

  • 模式说明页：规则摘要、常见术语、获胜条件。
  • 分工模板：可复制的「岗位检查表」与推荐构筑列表。

数据与维护

  • 公会向内容更新频率高，建议页脚统一「适用版本」与讨论区链接占位。
关键词池

6

每日清单

意图是「打开就干活」：用最低决策成本列出今日必做、可做、可鸽，提高回访与书签率。

典型搜索意图

  • 每天上线先做哪几项收益最高；周常与日常如何穿插。
  • 材料与代币：本周目标装备 / 技能还差哪些输入。
  • 与活动表联动：今天有没有必参加窗口。

本站如何承接

  • 每日待办页用本地存储记住勾选状态，适合固定开浏览器的一批用户。
  • 条目与构筑、副本、Boss 页交叉引用（「做完日常再去某活动」）。

详情页建议（SEO 骨架）

  • 「周循环」说明页：新号 / 老号不同优先级的两套清单。
  • 材料图鉴轻量页：只列获取途径与主要消耗去向。

数据与维护

  • 清单随版本增减条目；大改时在页顶公告一行「本周清单有更新」。
